# RadClean 平台启动教程

## 环境要求

- Python 3.10+
- 项目路径：`D:/孵化项目/radclean`
- 依赖已安装在全局 Python 环境中

---

## 一、启动后端服务器

打开终端（CMD / PowerShell / Git Bash），执行：

```bash
cd D:/孵化项目/radclean
python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000
```

启动成功后会看到：

```
INFO:     Started server process [xxxxx]
INFO:     Waiting for application startup.
INFO:     Application startup complete.
INFO:     Uvicorn running on http://0.0.0.0:8000
```

**保持终端窗口打开**，关闭窗口即停止服务器。

---

## 二、打开前端页面

直接用浏览器打开：

```
D:\孵化项目\radclean\frontend\index.html
```

或在浏览器地址栏输入：

```
file:///D:/孵化项目/radclean/frontend/index.html
```

---

## 三、停止服务器

按 `Ctrl + C` 终止终端中的 uvicorn 进程。

如果终端已关闭，可通过以下命令强制停止：

```bash
# PowerShell / CMD
netstat -ano | findstr :8000
taskkill /F /PID <PID>
```

或用一条命令：

```bash
python -c "import subprocess,re;r=subprocess.run('netstat -ano',capture_output=True,text=True);[subprocess.run(['taskkill','/F','/PID',l.split()[-1]]) for l in r.stdout.split('\n') if ':8000' in l and 'LISTENING' in l]"
```

---

## 四、重启服务器（代码修改后）

```bash
cd D:/孵化项目/radclean

# 1. 停掉旧进程
python -c "import subprocess;r=subprocess.run('netstat -ano',capture_output=True,text=True);[subprocess.run(['taskkill','/F','/PID',l.split()[-1]]) for l in r.stdout.split('\n') if ':8000' in l and 'LISTENING' in l]"

# 2. 启动新进程
python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000
```

可以写成一个批处理文件（参见第五节）。

---

## 五、一键启动脚本（可选）

在项目根目录创建 `start_server.bat`：

```batch
@echo off
chcp 65001 >nul
echo ========================================
echo   RadClean 后端服务器
echo ========================================
echo.
echo 正在停止旧进程...
python -c "import subprocess;r=subprocess.run('netstat -ano',capture_output=True,text=True);[subprocess.run(['taskkill','/F','/PID',l.split()[-1]]) for l in r.stdout.split('\n') if ':8000' in l and 'LISTENING' in l]" 2>nul
echo.
echo 正在启动服务器...
cd /d D:\孵化项目\radclean
python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000
pause
```

双击 `start_server.bat` 即可启动。关闭窗口即停止。

## 六、常见问题

### 端口被占用

```
ERROR: [Errno 10048] error while attempting to bind on address ('0.0.0.0', 8000)
```

**解决**：执行第四节第 1 步停掉旧进程，再启动。

### 页面打不开 / API 报错

确认后端服务器正在运行（终端中显示 `Uvicorn running on...`）。

### 前端 ECharts 不显示

确认 `frontend/js/vendor/echarts.min.js` 文件存在。如果缺失，从 npmmirror.com 下载：

```bash
cd D:/孵化项目/radclean/frontend
npm install echarts --registry=https://registry.npmmirror.com
copy node_modules\echarts\dist\echarts.min.js js\vendor\echarts.min.js
```

### LLM 增强报错

需要配置 API Key。复制 `backend/.env.example` 为 `backend/.env` 并填入你的 API Key：

```
LLM_API_KEY=你的key
```

支持任何兼容 OpenAI 协议的 API 服务（DeepSeek、OpenAI 等）。

未配置时 LLM 增强按钮会提示 "API key not configured"，不影响其他功能。

## 七、开发模式（自动重载）

代码修改后不想手动重启？加上 `--reload`：

```bash
python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000 --reload
```

修改 `backend/` 目录下的 `.py` 文件后服务器会自动重启。注意：`--reload` 适合开发调试，但不稳定（偶尔不触发重载），推荐演示时使用普通模式。
